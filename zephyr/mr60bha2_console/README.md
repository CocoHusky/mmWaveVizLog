# MR60BHA2 Zephyr Console

Zephyr firmware for the XIAO ESP32-C6 and Seeed MR60BHA2 sensor stack. This
path is the maintained runtime in this repository and keeps the sample schema,
dashboard behavior, LED control, and OTA flow aligned with the Arduino bring-up
reference.

## Runtime Goals

- boot on XIAO ESP32-C6
- decode MR60BHA2 frames from UART
- maintain bounded sample state
- stream schema-compatible JSON Lines over USB serial
- expose the local dashboard over Wi-Fi
- support LED control and threshold rules
- read ambient light from a BH1750 over I2C
- support Wi-Fi OTA using signed MCUboot images
- keep parser tests available for host-side validation

## Hardware Contract

The Zephyr target preserves the current Arduino wiring contract:

| Signal | GPIO |
| --- | ---: |
| MR60BHA2 UART RX | `17` |
| MR60BHA2 UART TX | `16` |
| WS2812 RGB LED | `1` |
| BH1750 I2C SDA | `22` |
| BH1750 I2C SCL | `23` |
| BH1750 I2C address | `0x23` |

## Local Zephyr Workspace

This repo keeps the application code in `zephyr/mr60bha2_console/` and the
downloaded Zephyr source tree in `zephyr/workspace/`.

`zephyr/workspace/` is ignored by Git because it is generated by `west` and is
not part of the application source.

### 1. Make `west` available

If `west` installed into `~/.local/bin`, add it to your shell path:

```sh
export PATH="$HOME/.local/bin:$PATH"
west --version
```

To make that permanent:

```sh
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
west --version
```

### 2. Create the local Zephyr workspace

Run these steps from the repository root. They create the local Zephyr checkout
that the app build depends on.

1. Go to the repository root.

   ```sh
   cd /Users/username/Documents/GitHub/mmWaveVizLog
   ```

   This puts you at the top of the project before creating the Zephyr workspace.

2. Create the workspace directory and enter it.

   ```sh
   mkdir -p zephyr/workspace
   cd zephyr/workspace
   ```

   This is where `west` will place the Zephyr source tree and modules.

3. Initialize `west`.

   ```sh
   west init
   ```

   This creates the workspace metadata and links it to the Zephyr manifest.

4. Download the Zephyr repositories.

   ```sh
   west update
   ```

   This fetches Zephyr, modules, and the required dependencies.

5. Export Zephyr package metadata for the local shell.

   ```sh
   west zephyr-export
   ```

   This makes the Zephyr Python packages visible to the current environment.

6. Install the Zephyr Python requirements.

   ```sh
   python3 -m pip install --user -r zephyr/scripts/requirements.txt
   ```

   This installs the Python tooling Zephyr needs for builds and scripts.

After this, the local tree should look like:

```text
zephyr/
  mr60bha2_console/   # this repo's application code
  workspace/          # local Zephyr checkout, ignored by Git
```

### 3. Install the build tools

Zephyr needs `cmake` and `ninja` in addition to `west`.
The Espressif build also needs `esptool` on your PATH.

If you have Homebrew:

```sh
brew install cmake ninja
cmake --version
ninja --version
```

If you do not have Homebrew, use conda:

```sh
conda install -c conda-forge cmake ninja
cmake --version
ninja --version
```

Install the ESP32 flashing tool:

```sh
python3 -m pip install --user 'esptool>=5.0.2'
esptool version
```

If the tools are still not found, make sure the same shell used by VS Code can see them:

```sh
which cmake
which ninja
echo $PATH
```

### 4. Install the Zephyr SDK

From the Zephyr repository in this workspace, install the SDK once:

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog/zephyr/workspace/zephyr
west sdk install --version 0.17.4 -t riscv64-zephyr-elf
```

If you install the SDK into a custom location, set `ZEPHYR_SDK_INSTALL_DIR` to that directory before building.

For command-line builds, explicitly select the Zephyr toolchain:

```sh
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
```

### 5. Build from inside the workspace

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog/zephyr/workspace
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
west build -b xiao_esp32c6/esp32c6/hpcore ../mr60bha2_console
```

### 6. Build from the repository root

If you prefer to run commands from the repository root, set `ZEPHYR_BASE` first:

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog
export ZEPHYR_BASE="$PWD/zephyr/workspace/zephyr"
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
west build -b xiao_esp32c6/esp32c6/hpcore zephyr/mr60bha2_console
```

If the board name is not available in your Zephyr checkout:

```sh
west boards | rg "xiao|esp32c6|seeed"
```

## Serial Validation

Open the device serial port at `115200`.

```sh
screen /dev/cu.usbmodem14201 115200
```

Expected output is one JSON object per line:

```json
{"type":"sample","heart_rate":null,"breath_rate":null,"distance":null,"raw_distance":null,"total_phase":null,"breath_phase":null,"heart_phase":null,"presence":false,"presence_valid":false,"presence_source":"waiting","target_valid":false,"target_source":"waiting","people_count":0,"target_count":0,"target_x":null,"target_y":null,"target_distance":null,"target_angle":null,"target_speed":null,"targets":[],"light":null,"light_ready":false,"console_fw":"Zephyr 2.2.1","firmware_valid":false,"firmware_raw":0,"firmware_project":0,"firmware_major":0,"firmware_sub":0,"firmware_modified":0,"led_r":0,"led_g":0,"led_b":0,"frame":0,"last_radar_ms":0,"uptime_ms":0}
```

Validation checklist:

1. Device boots without resetting repeatedly.
2. JSON Lines appear once per second.
3. `frame` increments.
4. `uptime_ms` increments.
5. Every line parses as JSON.
6. The parsed object validates against `protocol/sample.schema.json`.

## Parser Tests

```sh
west build -b native_sim zephyr/mr60bha2_console/tests/mr60bha2_parser
west twister -T zephyr/mr60bha2_console/tests
```

The parser test currently verifies initialization, empty input handling, partial frame handling, and MR60BHA2 frame envelope detection. Real captured radar frames should be added next.

If the BH1750 is present, the `light` and `light_ready` fields should populate after boot; otherwise they remain `null`/`false`.

## Hardware Bring-Up Notes

- The current board target is `xiao_esp32c6/esp32c6/hpcore`.
- UART, LED, and BH1750 wiring are defined in the board overlay.
- The serial sample stream and web dashboard use the same JSON schema as the
  other firmware paths in this repository.
