# mmWaveVizLog Zephyr Runtime

Zephyr firmware for the XIAO ESP32-C6 and Seeed MR60BHA2 sensor stack. This is the maintained product/runtime firmware path in this repository.

The Arduino quick-start bring-up reference lives at `arduino/mmwavevizlog-quickstart/`.

## Runtime Goals

- boot on XIAO ESP32-C6
- decode MR60BHA2 frames from UART
- maintain bounded sample state
- stream schema-compatible JSON Lines over USB serial
- expose the local dashboard over Wi-Fi
- support LED control and threshold rules
- read ambient light from a BH1750 over I2C
- support Wi-Fi OTA using signed MCUboot images
- keep parser tests available for host-side validation

## Hardware Contract

| Signal | GPIO |
| --- | ---: |
| MR60BHA2 UART RX | `17` |
| MR60BHA2 UART TX | `16` |
| WS2812 RGB LED | `1` |
| BH1750 I2C SDA | `22` |
| BH1750 I2C SCL | `23` |
| BH1750 I2C address | `0x23` |

## Versioning

The Zephyr app version is stored in `VERSION`. Release tags use `vA.B.C`, where `A` is major, `B` is minor, and `C` is patch. The first aligned release target is `v0.1.0`.

## Local Zephyr Workspace

This repo keeps the application code in `zephyr/mmwavevizlog-runtime/` and the downloaded Zephyr source tree in `zephyr/workspace/`.

`zephyr/workspace/` is ignored by Git because it is generated by `west` and is not part of the application source.

### 1. Make `west` available

```sh
export PATH="$HOME/.local/bin:$PATH"
west --version
```

To make that permanent:

```sh
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
west --version
```

### 2. Create the local Zephyr workspace

Run these steps from the repository root:

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog
mkdir -p zephyr/workspace
cd zephyr/workspace
west init
west update
west zephyr-export
python3 -m pip install --user -r zephyr/scripts/requirements.txt
```

After this, the local tree should look like:

```text
zephyr/
  mmwavevizlog-runtime/  # this repo's application code
  workspace/             # local Zephyr checkout, ignored by Git
```

### 3. Install the build tools

Zephyr needs `cmake` and `ninja`; Espressif builds also need `esptool`.

```sh
brew install cmake ninja
python3 -m pip install --user 'esptool>=5.0.2'
```

### 4. Install the Zephyr SDK

From the Zephyr repository in this workspace, install the SDK once:

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog/zephyr/workspace/zephyr
west sdk install --version 0.17.4 -t riscv64-zephyr-elf
```

For command-line builds, explicitly select the Zephyr toolchain:

```sh
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
```

### 5. Build from inside the workspace

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog/zephyr/workspace
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
west build -b xiao_esp32c6/esp32c6/hpcore ../mmwavevizlog-runtime
```

### 6. Build from the repository root

```sh
cd /Users/username/Documents/GitHub/mmWaveVizLog
export ZEPHYR_BASE="$PWD/zephyr/workspace/zephyr"
export ZEPHYR_TOOLCHAIN_VARIANT=zephyr
west build -b xiao_esp32c6/esp32c6/hpcore zephyr/mmwavevizlog-runtime
```

If the board name is not available in your Zephyr checkout:

```sh
west boards | rg "xiao|esp32c6|seeed"
```

## Flash

```sh
west flash
```

## Serial Validation

Open the device serial port at `115200`.

```sh
screen /dev/cu.usbmodem14201 115200
```

Expected output is one JSON object per line and should validate against `protocol/sample.schema.json`.

## Parser Tests

Required fast parser path:

```sh
west build -b native_sim zephyr/mmwavevizlog-runtime/tests/mr60bha2_parser
./build/zephyr/zephyr.exe
```

Advisory Twister path:

```sh
west twister -T zephyr/mmwavevizlog-runtime/tests --platform native_sim
```

The CI keeps the direct parser build/run as a required check. Twister is useful for Zephyr harness coverage, but it is advisory until the harness is hardened.

## Release Artifacts

Tagged releases build the real XIAO ESP32-C6 Zephyr firmware and publish the signed binary alongside the ELF, map file, native simulator parser executable, protocol schema, and protocol examples.
